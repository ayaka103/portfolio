# Code Review: セキュリティチェックリスト

セキュリティ上の脆弱性・リスクに関するチェック項目。

## チェック項目

### 1. 機密情報のハードコード

- [ ] メールアドレス・APIキー・パスワードがソースコードに直書きされていないか
- [ ] Git 履歴に機密情報が残っていないか
- [ ] 環境変数で管理すべき値がハードコードされていないか

**実例（ポートフォリオレビュー）**: `href="mailto:xxx@gmail.com"` が JSX に直書きされ、スパムボットにクロールされるリスクがあった。`data.ts` への集約やコンタクトフォームサービスの利用を推奨。

### 2. XSS (Cross-Site Scripting)

- [ ] ユーザー入力が適切にサニタイズされているか
- [ ] `dangerouslySetInnerHTML` が使用されていないか（使用されている場合は理由を確認）
- [ ] URL パラメータが直接 DOM に挿入されていないか

### 3. OWASP Top 10

- [ ] SQL インジェクションのリスクがないか
- [ ] CSRF 対策が実装されているか（フォーム送信がある場合）
- [ ] 不適切なアクセス制御がないか
- [ ] セキュリティヘッダーが適切に設定されているか

### 4. 依存関係

- [ ] `package.json` に脆弱性のある依存パッケージが含まれていないか
- [ ] 不要な依存パッケージが含まれていないか
- [ ] `npm audit` で警告が出ていないか

### 5. 静的サイト固有

- [ ] 静的エクスポート時に機密情報が HTML に埋め込まれていないか
- [ ] `NEXT_PUBLIC_` 環境変数に機密情報が含まれていないか
- [ ] ソースマップが本番環境で無効化されているか
