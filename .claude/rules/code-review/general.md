# Code Review: 一般品質チェックリスト

コードの品質・保守性・設計に関するチェック項目。

## チェック項目

### 1. コード重複

- [ ] 同一・類似のコードが複数ファイルに存在しないか
- [ ] コピペで作られた重複コンポーネント・ページがないか
- [ ] 共通ロジックが適切に抽出・共有されているか

**実例（ポートフォリオレビュー）**: `app/page.tsx` と `app/portfolio/page.tsx` が一字一句同じ内容で完全重複していた。片方を修正しても片方に反映されず、デグレードの温床になる。

### 2. 命名規則

- [ ] 変数・関数名が意味を正確に表しているか
- [ ] プロジェクト内で一貫した命名規則が使われているか
- [ ] Boolean 変数は `is`, `has`, `should` などのプレフィックスを持つか

### 3. 型設計

- [ ] 型定義がデータの実態を正しく反映しているか
- [ ] Optional フィールドが適切に `?` 付きで定義されているか
- [ ] `any` 型の使用が回避されているか
- [ ] 今後拡張が必要なフィールドが漏れていないか

**実例（ポートフォリオレビュー）**: `Project` 型に `url` フィールドがなく、プロジェクトへの外部リンク導線が作れない設計だった。

### 4. データ管理

- [ ] ハードコードされた値が定数ファイルに一元管理されているか
- [ ] 同じ値が複数箇所にベタ書きされていないか
- [ ] 設定値やマジックナンバーに名前が付けられているか

**実例（ポートフォリオレビュー）**: メールアドレスが JSX 内に直書きされ、変更時に全ファイル検索が必要になっていた。

### 5. ファイル構成

- [ ] 不要なファイルが存在しないか
- [ ] ファイルの責務が適切に分離されているか
- [ ] データとビューが分離されているか（データは `data.ts` 等に集約）
