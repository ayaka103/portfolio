# Code Review: アクセシビリティチェックリスト

WAI-ARIA、キーボード操作、モバイル対応に関するチェック項目。

## チェック項目

### 1. モーダル / ダイアログ

- [ ] `role="dialog"` と `aria-modal="true"` が設定されているか
- [ ] `aria-labelledby` でダイアログタイトルと紐付けられているか
- [ ] `Escape` キーでダイアログを閉じられるか
- [ ] ダイアログを開いた際にフォーカスがダイアログ内に移動するか
- [ ] ダイアログを閉じた際にフォーカスがトリガー要素に戻るか
- [ ] 閉じるボタンに `aria-label` が設定されているか

**実例（ポートフォリオレビュー）**: `ProjectCard.tsx` のモーダルに `role="dialog"`, `aria-modal`, `aria-labelledby` がなく、`Escape` キーでの閉じる操作も未実装だった。

### 2. ナビゲーション

- [ ] モバイル幅でナビゲーションの代替手段（ハンバーガーメニュー等）があるか
- [ ] `nav` 要素に `aria-label` が設定されているか
- [ ] スキップナビゲーション（「メインコンテンツへ」リンク）があるか
- [ ] キーボードでの Tab 順序が論理的か

**実例（ポートフォリオレビュー）**: `hidden md:flex` でナビが消えるが、モバイル向けの代替ナビゲーションが未実装だった。

### 3. 画像・メディア

- [ ] `img` / `Image` に意味のある `alt` テキストが設定されているか
- [ ] 装飾的な画像には `alt=""` が設定されているか
- [ ] 動画にキャプションがあるか

### 4. フォーム・インタラクション

- [ ] フォーム要素に `label` が関連付けられているか
- [ ] エラーメッセージが `aria-describedby` で関連付けられているか
- [ ] ボタンに明確なテキストまたは `aria-label` があるか
- [ ] クリック可能な要素が `button` または `a` タグを使用しているか（`div` + `onClick` を避ける）

### 5. 言語・色

- [ ] `<html lang="ja">` が正しく設定されているか
- [ ] 色のコントラスト比が WCAG AA 基準（4.5:1）を満たしているか
- [ ] 色だけに依存した情報伝達をしていないか
