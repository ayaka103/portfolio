# レビュー出力テンプレート

以下のMarkdown形式で出力する。

```markdown
## コードレビュー結果

### レビュー対象

- `ファイルパス1`
- `ファイルパス2`
- ...

---

### サマリー

[プロジェクト全体の品質を2〜3文で総括する。良い点と主要な問題を簡潔にまとめる]

---

### 指摘事項

#### 1. [指摘タイトル]（[カテゴリ名]）

**ファイル**: `ファイルパス` (行 XX-YY)

**問題**: [何が問題なのかを具体的に説明する]

**理由**: [なぜ問題なのか、どのような影響があるのかを説明する]

**改善案**:

\`\`\`tsx
// 具体的なコード例を示す
\`\`\`

---

[指摘2, 3, ... を同じフォーマットで続ける]

---

### 良い点

- [良い設計・実装パターン1の説明]
- [良い設計・実装パターン2の説明]
- [良い設計・実装パターン3の説明]

---

### 総合評価

**優先度順の対応推奨事項:**

1. **即対応が必要**: 指摘X ([タイトル]) — [理由]
2. **高優先度**: 指摘Y ([タイトル]) — [理由]
3. **中優先度**: 指摘Z ([タイトル])
4. **低優先度**: 指摘W ([タイトル])
```

## テンプレートルール

- 指摘事項は優先度順（即対応 → 高 → 中 → 低）に並べる
- 各指摘には必ず「ファイル:行番号」「問題」「理由」「改善案」を含める
- 改善案には実行可能なコード例を提示する
- 良い点は最低3つ記載する
- 総合評価では優先度別に対応順序を明示する
- カテゴリ名は以下のいずれか: バグ, セキュリティ, パフォーマンス, アクセシビリティ, SEO, 保守性, 型設計, UX, CSS
