# File Summarizer

指定されたファイルを読み込み、構造化された要約を出力する。

## Usage

```
/summarize <file-path>
```

引数 `$ARGUMENTS` からファイルパスを取得する。引数がない場合は AskUserQuestion でファイルパスを確認する。

## Workflow

1. `$ARGUMENTS` からファイルパスを取得する（空なら AskUserQuestion で確認）
2. Read ツールでファイルを読み込む
3. 内容を分析し、以下のフォーマットで要約を出力する

## Output Format

以下のMarkdown形式で出力する:

```markdown
## [タイトル]
ファイルの内容を最も的確に表す簡潔なタイトル

### トピック
1. **[トピック1の見出し]**: [1〜2文の説明]
2. **[トピック2の見出し]**: [1〜2文の説明]
3. **[トピック3の見出し]**: [1〜2文の説明]

### 結論
[ファイル全体の要点を2〜3文でまとめる]

### 改善提案
1. **[提案1の見出し]**: [具体的な改善内容を1〜2文で]
2. **[提案2の見出し]**: [具体的な改善内容を1〜2文で]
3. **[提案3の見出し]**: [具体的な改善内容を1〜2文で]
```

## Rules

- タイトルはファイルの主題を端的に表現する
- トピックはファイル内の最も重要な3つのポイントを抽出する
- 結論はファイル全体を俯瞰した要約とする
- 日本語のファイルは日本語で、英語のファイルは英語で要約する
- コードファイルの場合は、そのコードの目的・主要機能・設計方針を要約する
- 改善提案はドキュメントの品質・構成・網羅性・読みやすさの観点から具体的に提示する
- 改善提案はコードファイルなら保守性・可読性・パフォーマンス、文書なら明確さ・構造・不足情報の観点で提示する
- 要約・改善提案以外の余計なコメントや前置きは出力しない
